# Results directory

Results are grouped by experiment family so unrelated baselines, teachers, and
distillation runs are not mixed together.

## `01_base_model/`

Mean-fusion SuperLightNet results and direct ablations:

- `leakage_safe/` — original patient-split training logs and evaluations.
- `leakage_safe_fixed/` — corrected modality-mask evaluation sweep.
- `ablation_no_rmd/` — RMD-disabled ablation and its evaluations.
- `ablation_no_rmd_manual/` — earlier manual ablation run.

## `02_kd_old_swin/`

Historical Swin-KD work using the older fold teacher:

- `swin_kd_v1/` — training and selection records.
- `swin_kd_v1_sweep/` — 15-subset test sweep.
- `swin_teacher_sanity/` — teacher provenance evaluation.
- `teacher_reorder_check/` — historical channel-reorder diagnostics.

These results are retained for provenance and must not be confused with the
clean-teacher experiment.

## `03_self_kd/`

Self-distillation experiments, including cancelled runs, V2–V5, seed
replications, control runs, and their modality sweeps.

## `04_kd_clean_swin/`

Leakage-safe clean Swin teacher and KD results:

- `swin_teacher_clean_test/` — clean teacher held-out evaluation.
- `swin_kd_clean_A/` — mean-fusion student distilled from the clean teacher.
- `swin_kd_clean_A_sweep/` — generated by the clean-A sweep evaluator.

## `05_analysis/`

Cross-experiment paper tables and analysis inputs.

Runtime checkpoints remain under `checkpoints/`; this directory contains logs,
per-case evaluations, aggregate JSON files, and analysis artifacts only.

## Log naming convention

Every training artifact includes its experiment/version prefix:

- `<experiment>_training_log.csv` — epoch-level training and validation metrics.
- `<experiment>_training_batch_log.csv` — batch-level optimization metrics.
- `<experiment>_validation_tradeoff.csv` — checkpoint-selection trade-off history.

For example, the clean mean-fusion KD run uses
`swin_kd_clean_A_training_log.csv`, while the V3 self-distillation run uses
`self_distill_v3_training_log.csv`.
